awk - Język przetwarzania wzorców i tekstu

Opis

awk to język programowania przeznaczony do przetwarzania tekstowych plików danych, szczególnie przydatny dla danych strukturyzowanych w kolumnach.

Składnia

awk [OPCJE] 'program' [plik...]
awk [OPCJE] -f program-file [plik...]

Parametry

  • -F fs - Separator pól (field separator)
  • -v var=value - Przypisz zmienną
  • -f program-file - Czytaj program z pliku
  • -- - Koniec opcji

Struktura programu

BEGIN { działania_początkowe }
/wzorzec/ { działania }
END { działania_końcowe }

Zmienne wbudowane

  • $0 - Cała linia
  • $1, $2, ... - Pola 1, 2, …
  • $NF - Ostatnie pole
  • NF - Liczba pól w linii
  • NR - Numer bieżącej linii (all files)
  • FNR - Numer linii w bieżącym pliku
  • FS - Separator pól (input)
  • OFS - Separator pól (output)
  • RS - Separator rekordów (input)
  • ORS - Separator rekordów (output)
  • FILENAME - Nazwa bieżącego pliku

Przykłady użycia

Podstawowe

  1. Drukuj całą linię
awk '{print}' file.txt
  1. Drukuj pierwsze pole
awk '{print $1}' file.txt
  1. Drukuj ostatnie pole
awk '{print $NF}' file.txt
  1. Wiele pól
awk '{print $1, $3}' file.txt
  1. Z własnym separatorem
awk -F: '{print $1}' /etc/passwd
  1. OFS - output separator
awk 'BEGIN {OFS=","} {print $1, $2}' file.txt

Filtrowanie

  1. Linie pasujące do wzorca
awk '/error/' log.txt
  1. Warunek na pole
awk '$3 > 100' data.txt
  1. Kombinacja warunków
awk '$3 > 100 && $1 == "active"' data.txt
  1. Negacja
awk '$1 != "root"' /etc/passwd
  1. Regex na pole
awk '$2 ~ /^test/' file.txt
  1. NOT matching regex
awk '$2 !~ /^test/' file.txt

Operacje matematyczne

  1. Suma kolumny
awk '{sum += $1} END {print sum}' numbers.txt
  1. Średnia
awk '{sum += $1; count++} END {print sum/count}' numbers.txt
  1. Min/Max
awk 'NR==1 {max=$1} $1>max {max=$1} END {print max}' numbers.txt
  1. Liczba linii spełniających warunek
awk '$3 > 100 {count++} END {print count}' data.txt

BEGIN i END

  1. Nagłówek i stopka
awk 'BEGIN {print "Start"} {print} END {print "End"}' file.txt
  1. Inicjalizacja zmiennych
awk 'BEGIN {sum=0} {sum += $1} END {print sum}' numbers.txt
  1. Ustawienie separatora
awk 'BEGIN {FS=":"} {print $1}' /etc/passwd

Zaawansowane

  1. Numeracja linii (niestandardowa)
awk '{print NR, $0}' file.txt
  1. Konkretna linia
awk 'NR==5' file.txt
  1. Zakres linii
awk 'NR>=10 && NR<=20' file.txt
  1. Co druga linia
awk 'NR%2==0' file.txt
  1. Usuń puste linie
awk 'NF > 0' file.txt
  1. Unikalne linie
awk '!seen[$0]++' file.txt
  1. Zliczanie unikalnych wartości
awk '{count[$1]++} END {for (i in count) print i, count[i]}' file.txt
  1. Formatowanie wyjścia
awk '{printf "%-10s %5d\n", $1, $2}' file.txt
  1. String operations
awk '{print tolower($1)}' file.txt
awk '{print toupper($1)}' file.txt
awk '{print length($1)}' file.txt
  1. Substr
awk '{print substr($1, 1, 3)}' file.txt
  1. Split
awk '{split($1, arr, "-"); print arr[1]}' file.txt

Operatory

Arytmetyczne: + - * / % ^ ** Porównania: < <= == != >= > Logiczne: && || ! String: ~ (matches), !~ (doesn’t match) Przypisanie: = += -= *= /= %= ^=

Funkcje

String: - length(s) - Długość - tolower(s) - Do małych liter - toupper(s) - Do wielkich liter - substr(s, i, n) - Substring - index(s, t) - Pozycja t w s - split(s, a, fs) - Dzieli string - sub(r, s, t) - Zamień pierwsze - gsub(r, s, t) - Zamień wszystkie

Math: - int(x) - Integer part - sqrt(x) - Pierwiastek - sin(x), cos(x), atan2(y,x) - Trygonometria - rand() - Random [0,1) - srand(x) - Seed dla rand

Przykłady praktyczne

Analiza logów:

awk '$9 == 404' access.log
awk '{print $1}' access.log | sort | uniq -c | sort -rn

CSV processing:

awk -F, '{print $2, $4}' data.csv

Obliczenia:

ls -l | awk '{sum += $5} END {print sum}'

Wskazówki

  1. BEGIN dla inicjalizacji - Ustawiaj zmienne i separatory
  2. END dla podsumowań - Idealne dla sum, średnich
  3. $0 vs NF - $0 to cała linia, NF to liczba pól
  4. Tablice asocjacyjne - Bardzo przydatne dla agregacji
  5. printf dla formatowania - Lepszy formatting niż print

Powiązane komendy

  • sed - Stream editing
  • grep - Pattern matching
  • cut - Cut fields
  • perl - Bardziej zaawansowany

Zwracane wartości

  • 0 - Sukces
  • 1+ - Błąd

break - Przerwanie

Opis

Wychodzi z pętli

Przykłady użycia

  1. Exit loop
while true; do if [ $cond ]; then break; fi; done
  1. For loop
for i in {1..100}; do if [ $i -eq 50 ]; then break; fi; done
  1. Inner loop
for x in *; do for y in *; do if [ $cond ]; then break; fi; done; done
  1. Break level
for x in *; do for y in *; do break 2; done; done
  1. After error
while read line; do process || break; done
  1. Found item
for item in "${array[@]}"; do if [ "$item" = "target" ]; then break; fi; done
  1. Limit reached
i=0; while true; do if [ $i -ge 1000 ]; then break; fi; ((i++)); done
  1. User quit
while true; do read cmd; if [ "$cmd" = "quit" ]; then break; fi; done
  1. File found
for dir in */; do if [ -f $dir/target ]; then echo found; break; fi; done
  1. Network up
while true; do if ping -c 1 host; then break; fi; sleep 1; done
  1. Multiple breaks
while true; do case $input in q) break;; esac; done
  1. With cleanup
while true; do work || { cleanup; break; }; done
  1. Outer loop
for x in *; do for y in *; do [ $found ] && break 2; done; done
  1. Exit on signal
trap 'break' INT; while true; do work; done
  1. Conditional
for i in *; do [ -d $i ] && break; done
  1. Search stop
for f in $(find .); do grep -q pattern $f && break; done
  1. Resource limit
while true; do [ $(df -h | awk '$6=="/" {print $5}' | tr -d %) -gt 90 ] && break; done
  1. Time limit
START=$SECONDS; while true; do [ $((SECONDS-START)) -gt 60 ] && break; done
  1. Error count
ERR=0; while [ $ERR -lt 3 ]; do cmd || ((ERR++)); [ $ERR -eq 3 ] && break; done
  1. Parse complete
while read line; do [ "$line" = "END" ] && break; process $line; done

cat - Konkatenacja i wyświetlanie plików

Opis

Komenda cat (concatenate) łączy pliki i wyświetla je na standardowym wyjściu. Jest jedną z najprostszych, ale najczęściej używanych komend w systemach Unix/Linux.

Składnia

cat [OPCJE] [PLIK...]

Parametry i opcje

  • -A, --show-all - Równoważne -vET
  • -b, --number-nonblank - Numeruje niepuste linie
  • -e - Równoważne -vE
  • -E, --show-ends - Wyświetla $ na końcu każdej linii
  • -n, --number - Numeruje wszystkie linie wyjścia
  • -s, --squeeze-blank - Redukuje wielokrotne puste linie do jednej
  • -t - Równoważne -vT
  • -T, --show-tabs - Wyświetla TAB jako ^I
  • -u - (ignorowane, kompatybilność z UNIX)
  • -v, --show-nonprinting - Używa notacji ^ i M- dla znaków niedrukowalnych

Przykłady użycia

Podstawowe wyświetlanie

  1. Wyświetl zawartość pliku
cat plik.txt
  1. Wyświetl wiele plików
cat plik1.txt plik2.txt plik3.txt
  1. Konkatenacja plików
cat plik1.txt plik2.txt > polaczony.txt
  1. Dodaj do istniejącego pliku
cat nowy.txt >> istniejacy.txt
  1. Wyświetl z numerami linii
cat -n plik.txt
  1. Numeruj tylko niepuste linie
cat -b plik.txt
  1. Redukuj puste linie
cat -s plik.txt

Tworzenie plików

  1. Utwórz plik z treścią

Wpisz treść
Zakończ Ctrl+D
  1. Heredoc - tworzenie z blokiem tekstu

EOF
  1. Dodaj do pliku

EOF

Wyświetlanie specjalne

  1. Pokaż znaki końca linii

  1. Pokaż tabulatory

  1. Pokaż wszystkie znaki specjalne

  1. Bez pustych linii

Kombinacje z pipe

  1. Z grep

  1. Z wc do liczenia

  1. Tee dla podwójnego wyjścia

cd - Zmiana katalogu

Opis

Komenda cd (change directory) zmienia bieżący katalog roboczy. Jest to wbudowana komenda powłoki (shell builtin), nie zewnętrzny program. Pozwala nawigować po strukturze katalogów systemu plików.

Składnia

cd [OPCJE] [KATALOG]

Parametry i opcje

Opcje

  • -L - Podąża za linkami symbolicznymi (domyślne zachowanie)
  • -P - Używa fizycznej struktury katalogów, nie podąża za linkami symbolicznymi
  • -e - Jeśli nie można zmienić katalogu z -P, zwróć błąd
  • -@ - Przeglądaj atrybuty rozszerzone (macOS)

Specjalne symbole

  • ~ - Katalog domowy użytkownika
  • - - Poprzedni katalog roboczy
  • .. - Katalog nadrzędny
  • . - Bieżący katalog
  • / - Katalog główny (root)

Przykłady użycia

Podstawowe użycie

  1. Przejdź do katalogu domowego

# lub
cd ~
  1. Przejdź do konkretnego katalogu
cd /usr/local/bin
  1. Przejdź do katalogu nadrzędnego
cd ..
  1. Przejdź dwa poziomy wyżej
cd ../..
  1. Przejdź do poprzedniego katalogu
cd -
# Wyświetla ścieżkę poprzedniego katalogu
  1. Przejdź do katalogu głównego
cd /
  1. Przejdź do podkatalogu
cd Documents

Użycie ścieżek względnych

  1. Względem bieżącego katalogu

  1. Nawigacja w górę i w dół
cd ../inny-projekt/tests
  1. Pozostań w bieżącym katalogu

# Nie zmienia katalogu, ale może być użyteczne w skryptach

Użycie ze zmiennymi

  1. Użyj zmiennej HOME

  1. Użyj zmiennej PWD
cd $OLDPWD
# Równoważne cd -
  1. Katalog domowy innego użytkownika

Zaawansowane użycie

  1. Przejdź do katalogu ze spacjami w nazwie

# lub

  1. Przejdź za linkiem symbolicznym (domyślnie)

  1. Przejdź bez podążania za linkami

# Przechodzi do rzeczywistej lokalizacji
  1. Sprawdź czy katalog istnieje przed przejściem

  1. Przejdź i wykonaj komendę

  1. Przejdź, wykonaj i wróć

# Po zakończeniu pozostajesz w oryginalnym katalogu

  1. Przejdź do najnowszego katalogu

  1. Przejdź do katalogu z historii

# Ostatni katalog w historii katalogów (w niektórych powłokach)

chmod - Zmiana uprawnień plików

Opis

Komenda chmod (change mode) zmienia uprawnienia dostępu do plików i katalogów. Kontroluje kto może czytać, pisać i wykonywać pliki.

Składnia

chmod [OPCJE] MOD PLIK...
chmod [OPCJE] TRYB_OKTOWY PLIK...

Parametry i opcje

  • -c, --changes - Jak verbose, ale pokazuje tylko gdy wprowadzono zmiany
  • -f, --silent, --quiet - Ukrywa większość komunikatów błędów
  • -v, --verbose - Wyświetla diagnostykę dla każdego pliku
  • -R, --recursive - Zmienia pliki i katalogi rekursywnie
  • --reference=RPLIK - Użyj uprawnień RPLIK
  • --help - Pomoc
  • --version - Wersja

System uprawnień

Notacja numeryczna (oktalna)

  • 4 - read (r)
  • 2 - write (w)
  • 1 - execute (x)

Trzy cyfry: [właściciel][grupa][inni]

Notacja symboliczna

  • u - user (właściciel)
  • g - group (grupa)
  • o - others (inni)
  • a - all (wszyscy)

Operatory: - + - dodaj uprawnienie - - - usuń uprawnienie - = - ustaw dokładnie te uprawnienia

Uprawnienia: - r - read - w - write
- x - execute - X - execute tylko jeśli plik jest katalogiem lub ma już x - s - setuid/setgid - t - sticky bit

Przykłady użycia

Notacja numeryczna

  1. Pełny dostęp dla właściciela
chmod 700 plik.txt
# rwx------ (właściciel: czytaj, pisz, wykonuj)
  1. Standardowe uprawnienia pliku
chmod 644 plik.txt
# rw-r--r-- (właściciel: rw, grupa: r, inni: r)
  1. Standardowe uprawnienia katalogu
chmod 755 katalog
# rwxr-xr-x
  1. Wykonywalne tylko dla właściciela
chmod 744 skrypt.sh
# rwxr--r--
  1. Pełne uprawnienia dla wszystkich
chmod 777 plik.txt
# rwxrwxrwx (NIEBEZPIECZNE!)
  1. Brak uprawnień
chmod 000 plik.txt
# ---------
  1. Typowe dla skryptu
chmod 755 skrypt.sh
# rwxr-xr-x

Notacja symboliczna

  1. Dodaj uprawnienie wykonywania
chmod +x skrypt.sh
  1. Usuń uprawnienie zapisu dla innych
chmod o-w plik.txt
  1. Dodaj czytanie dla grupy
chmod g+r plik.txt
  1. Ustaw dokładne uprawnienia
chmod u=rwx,g=rx,o=r plik.txt
  1. Dodaj wykonywanie dla wszystkich
chmod a+x skrypt.sh
  1. Usuń wszystkie uprawnienia dla innych
chmod o= plik.txt
  1. Dodaj zapis dla właściciela i grupy
chmod ug+w plik.txt

Rekursywne zmiany

  1. Zmień uprawnienia rekursywnie
chmod -R 755 katalog/
  1. Rekursywnie z informacją
chmod -Rv u+x bin/
  1. X - execute tylko dla katalogów
chmod -R u+X *
# Dodaje x tylko dla katalogów, nie plików

Specjalne uprawnienia

  1. Setuid bit
chmod u+s program
# lub
chmod 4755 program
  1. Setgid bit
chmod g+s katalog
# lub
chmod 2755 katalog
  1. Sticky bit
chmod +t /tmp
# lub
chmod 1777 /tmp
  1. Kombinacja specjalnych
chmod 6755 plik  # setuid + setgid
chmod 7777 plik  # wszystkie specjalne + pełne uprawnienia

Zaawansowane

  1. Kopiuj uprawnienia z innego pliku
chmod --reference=wzorzec.txt cel.txt
  1. Zmień tylko jeśli plik ma określone uprawnienia
find . -perm 777 -exec chmod 755 {} \;
  1. Rekursywnie - katalogi 755, pliki 644
find . -type d -exec chmod 755 {} \;
find . -type f -exec chmod 644 {} \;
  1. Warunkowa zmiana
[ -f skrypt.sh ] && chmod +x skrypt.sh

  2. Wykonywanie katalogów - Katalog potrzebuje x do przejścia

cp - Kopiowanie plików i katalogów

Opis

Komenda cp (copy) kopiuje pliki i katalogi. Tworzy duplikat źródła w nowej lokalizacji, pozostawiając oryginał niezmieniony.

Składnia

cp [OPCJE] ŹRÓDŁO CEL
cp [OPCJE] ŹRÓDŁO... KATALOG

Parametry i opcje

Podstawowe opcje

  • -r, -R, --recursive - Kopiuj katalogi rekursywnie
  • -i, --interactive - Pytaj przed nadpisaniem
  • -f, --force - Wymuś nadpisanie bez pytania
  • -n, --no-clobber - Nie nadpisuj istniejących plików
  • -v, --verbose - Pokazuj co jest kopiowane
  • -u, --update - Kopiuj tylko nowsze pliki
  • -a, --archive - Tryb archiwum (zachowuje wszystko: -dR –preserve=all)
  • -l, --link - Twórz twarde dowiązania zamiast kopiować
  • -s, --symbolic-link - Twórz dowiązania symboliczne
  • -p - Zachowaj atrybuty (uprawnienia, właściciel, czas)
  • --preserve[=ATTR] - Zachowaj określone atrybuty
  • -d - Nie podążaj za linkami symbolicznymi, kopiuj jako linki
  • -b - Twórz backup nadpisywanych plików
  • -t, --target-directory=DIR - Kopiuj do katalogu DIR

Przykłady użycia

Podstawowe kopiowanie

  1. Kopiuj plik
cp plik.txt kopia.txt
  1. Kopiuj do katalogu
cp plik.txt /home/user/backup/
  1. Kopiuj wiele plików
cp plik1.txt plik2.txt plik3.txt /backup/
  1. Kopiuj z potwierdzeniem
cp -i plik.txt istniejacy.txt
# Zapyta przed nadpisaniem
  1. Kopiuj z informacją
cp -v plik.txt kopia.txt
# Output: 'plik.txt' -> 'kopia.txt'

Kopiowanie katalogów

  1. Kopiuj katalog rekursywnie
cp -r katalog kopia-katalogu
  1. Kopiuj zachowując atrybuty
cp -rp katalog backup-katalogu
  1. Tryb archiwum (pełna kopia)
cp -a katalog backup
# Zachowuje wszystko: uprawnienia, czasy, linki
  1. Kopiuj zawartość katalogu
cp -r katalog/. cel/
# Kopiuje zawartość, nie sam katalog
  1. Kopiuj bez nadpisywania
cp -rn source/ destination/

Zaawansowane opcje

  1. Kopiuj tylko nowsze pliki
cp -ru source/ backup/
  1. Twórz backup nadpisywanych
cp -b plik.txt istniejacy.txt
# Tworzy istniejący.txt~
  1. Własny suffix backupu
cp --backup=numbered plik.txt istniejacy.txt
  1. Twórz twarde dowiązania
cp -l plik.txt link-do-pliku.txt
  1. Twórz dowiązania symboliczne
cp -s /path/to/file symlink

Wzorce i wildcards

  1. Kopiuj wszystkie .txt
cp *.txt /backup/
  1. Kopiuj różne rozszerzenia
cp *.{txt,pdf,doc} /backup/
  1. Kopiuj z wykluczeniem
shopt -s extglob
cp !(*.tmp) /backup/
  1. Kopiuj ukryte pliki
cp -r .* /backup/
# lub
cp -r . .. /backup/  # Kopiuje wszystko

Kombinacje z innymi komendami

  1. Kopiuj z find
find . -name "*.log" -exec cp {} /backup/ \;
  1. Kopiuj z zachowaniem struktury
find . -name "*.txt" | cpio -pdm /backup/
  1. Kopiuj największe pliki
ls -S | head -5 | xargs -I {} cp {} /backup/
  1. Kopiuj z data stamp
cp plik.txt plik-$(date +%Y%m%d).txt
  1. Sync directories
rsync -av --progress source/ dest/
# Lepsze niż cp dla dużych kopii
  1. Kopiuj z paskiem postępu
rsync -ah --progress plik.txt /backup/
# lub używając pv
pv plik.txt > /backup/plik.txt

Częste scenariusze

Backup skrypty

# Incremental backup

  1. Slash na końcu - cp dir/ dest/ kopiuje zawartość, cp dir dest/ kopiuje cały katalog
  2. Domyślnie nie rekursywne - Musisz użyć -r dla katalogów
  3. Nadpisywanie - Domyślnie nadpisuje bez pytania (użyj -i)
  4. Zachowanie atrybutów - Użyj -a lub -p dla pełnej kopii
  5. Wydajność - Dla dużych kopii rozważ rsync
  6. Across filesystems - cp działa między różnymi systemami plików
  7. Uprawnienia - Domyślnie kopiuje zgodnie z umask, nie oryginalne

date - Wyświetlanie i ustawianie daty/czasu

Opis

Komenda date wyświetla lub ustawia datę i czas systemowy. Pozwala formatować wyjście na wiele sposobów i wykonywać obliczenia dat.

Składnia

date [OPCJE]... [+FORMAT]
date [-u|--utc|--universal] [MMDDhhmm[[CC]YY][.ss]]

Parametry i opcje

  • -d, --date=STRING - Wyświetl czas opisany przez STRING
  • -f, --file=PLIK - Jak –date dla każdej linii z PLIKU
  • -r, --reference=PLIK - Wyświetl czas ostatniej modyfikacji PLIKU
  • -R, --rfc-email - Format RFC 5322 (e-mail)
  • --rfc-3339=SPEC - Format RFC 3339 (date, seconds, ns)
  • -s, --set=STRING - Ustaw czas opisany przez STRING
  • -u, --utc, --universal - Użyj czasu UTC zamiast lokalnego
  • --help - Pomoc
  • --version - Wersja

Formaty daty

Podstawowe znaczniki

  • %Y - Rok (4 cyfry) - 2024
  • %y - Rok (2 cyfry) - 24
  • %m - Miesiąc (01-12)
  • %d - Dzień (01-31)
  • %H - Godzina 24h (00-23)
  • %I - Godzina 12h (01-12)
  • %M - Minuta (00-59)
  • %S - Sekunda (00-59)
  • %N - Nanosekundy
  • %p - AM/PM

Nazwy

  • %A - Pełna nazwa dnia (Monday)
  • %a - Skrócona nazwa dnia (Mon)
  • %B - Pełna nazwa miesiąca (January)
  • %b - Skrócona nazwa miesiąca (Jan)

Złożone

  • %F - Pełna data YYYY-MM-DD
  • %T - Pełny czas HH:MM:SS
  • %R - Czas HH:MM
  • %D - Data MM/DD/YY
  • %x - Data lokalna
  • %X - Czas lokalny
  • %c - Data i czas lokalny

Inne

  • %s - Sekundy od epoch (1970-01-01 00:00:00 UTC)
  • %u - Dzień tygodnia (1-7, 1=poniedziałek)
  • %w - Dzień tygodnia (0-6, 0=niedziela)
  • %j - Dzień roku (001-366)
  • %U - Numer tygodnia (00-53)
  • %V - Numer tygodnia ISO (01-53)
  • %Z - Strefa czasowa (np. CET)
  • %z - Offset strefy (+0100)

Przykłady użycia

Podstawowe wyświetlanie

  1. Data i czas (domyślny)
date
# Mon Jan 15 14:30:45 CET 2024
  1. Data w formacie YYYY-MM-DD
date +%Y-%m-%d
# 2024-01-15
  1. Czas HH:MM:SS
date +%H:%M:%S
# 14:30:45
  1. Data i czas ISO 8601
date +%Y-%m-%dT%H:%M:%S
# 2024-01-15T14:30:45
  1. UTC
date -u

Formatowanie custom

  1. Dla plików backup

# 20240115_143045
  1. Czytelny format

# Today is Monday, January 15, 2024
  1. Dla logów

# [2024-01-15 14:30:45]
  1. Unix timestamp
date +%s
# 1705328445
  1. Nazwa dnia i miesiąca

# Monday, January

Obliczenia dat

  1. Jutro

  1. Wczoraj

  1. Za 5 dni
date -d "+5 days"
# lub

  1. Konkretna data w przyszłości

  1. Dodaj godziny
date -d "+3 hours"
  1. Mieszane operacje
date -d "2 days ago 3 hours"

Z plikami

  1. Czas modyfikacji pliku

  1. Dodaj timestamp do pliku
mv log.txt log-$(date +%Y%m%d).txt
  1. Twórz katalog z datą
mkdir backup-$(date +%F)
  1. Nazwa pliku z pełnym timestamp
touch "report_$(date +%Y%m%d_%H%M%S).txt"

W skryptach

  1. Start time benchmark
START=$(date +%s)
# ... operacje ...

echo "Czas wykonania: $((END - START)) sekund"
  1. Logging
echo "$(date +"%Y-%m-%d %H:%M:%S") - Rozpoczęto backup" >> log.txt
  1. Conditional na podstawie daty

    echo "Dzień roboczy"
else
    echo "Weekend"
fi

grep - Wyszukiwanie wzorców w tekście

Opis

grep (Global Regular Expression Print) przeszukuje pliki lub strumienie w poszukiwaniu linii pasujących do wzorca. Jedno z najpotężniejszych narzędzi do przetwarzania tekstu w Unix/Linux.

Składnia

grep [OPCJE] WZORZEC [PLIK...]

Parametry i opcje

Wybór wzorca: - -E, --extended-regexp - ERE (rozszerzone wyrażenia) - -F, --fixed-strings - Literalne stringi (nie regex) - -G, --basic-regexp - BRE (podstawowe, domyślne) - -P, --perl-regexp - PCRE (Perl regex) - -e WZORZEC - Wielokrotne wzorce - -f PLIK - Wzorce z pliku - -i, --ignore-case - Ignoruj wielkość liter - -v, --invert-match - Odwróć dopasowanie - -w, --word-regexp - Całe słowa - -x, --line-regexp - Całe linie

Wyjście: - -c, --count - Liczba dopasowanych linii - -color[=WHEN] - Kolorowanie - -L, --files-without-match - Pliki BEZ dopasowań - -l, --files-with-matches - Pliki Z dopasowaniami - -m NUM - Maksimum NUM dopasowań - -n, --line-number - Numery linii - -H, --with-filename - Drukuj nazwy plików - -h, --no-filename - Ukryj nazwy plików - -o, --only-matching - Tylko dopasowana część - -q, --quiet - Cichy tryb (tylko exit status) - -s, --no-messages - Ukryj błędy

Kontekst: - -A NUM - NUM linii PO dopasowaniu - -B NUM - NUM linii PRZED dopasowaniem - -C NUM - NUM linii WOKÓŁ (before+after)

Katalogi: - -r, --recursive - Rekursywnie w katalogach - -R, --dereference-recursive - Jak -r + linki symboliczne - --include=WZORZEC - Tylko pliki pasujące - --exclude=WZORZEC - Pomiń pliki pasujące - --exclude-dir=WZORZEC - Pomiń katalogi

Przykłady użycia

Podstawowe wyszukiwanie

  1. Znajdź wzorzec w pliku

  1. Początek linii

  1. Koniec linii

  1. Dowolny znak

  1. Klasy znaków

  1. Rozszerzone regex (ERE)

Metaznaki regex

Podstawowe (BRE): - . - Dowolny znak - * - 0 lub więcej - ^ - Początek linii - $ - Koniec linii - [] - Klasa znaków - \ - Escape

Rozszerzone (ERE z -E): - + - 1 lub więcej - ? - 0 lub 1 - | - Alternatywa - () - Grupowanie - {n} - Dokładnie n - {n,} - Co najmniej n - {n,m} - Od n do m

Perl (PCRE z -P): - \d - Cyfra - \w - Znak słowa - \s - Whitespace - \b - Granica słowa

ls - Lista plików i katalogów

Opis

Komenda ls (list) wyświetla zawartość katalogów. Jest jedną z najczęściej używanych komend w systemach Unix/Linux, pozwalająca przeglądać pliki i katalogi w systemie plików.

Składnia

ls [OPCJE] [PLIK...]

Parametry i opcje

Podstawowe opcje

  • -a, --all - Wyświetla wszystkie pliki, włącznie z ukrytymi (zaczynającymi się od .)
  • -A, --almost-all - Jak -a, ale pomija . i ..
  • -l - Format długi (szczegółowy), pokazuje uprawnienia, właściciela, rozmiar, datę
  • -h, --human-readable - Wyświetla rozmiary w formacie czytelnym dla człowieka (K, M, G)
  • -d, --directory - Wyświetla informacje o katalogu zamiast jego zawartości
  • -R, --recursive - Rekursywne wyświetlanie podkatalogów
  • -r, --reverse - Odwrócona kolejność sortowania
  • -t - Sortuj według czasu modyfikacji (najnowsze pierwsze)
  • -S - Sortuj według rozmiaru pliku (największe pierwsze)
  • -i, --inode - Wyświetla numer i-węzła każdego pliku
  • -1 - Wyświetla jeden plik w linii

Formatowanie

  • -F, --classify - Dodaje wskaźnik typu pliku (/ dla katalogów, * dla wykonywalnych)
  • -G, --no-group - W formacie długim nie wyświetla grupy
  • -n, --numeric-uid-gid - Wyświetla numeryczne UID i GID zamiast nazw
  • --color[=WHEN] - Kolorowanie wyjścia (auto, always, never)
  • --full-time - Pełny format czasu (z nanosekundami)
  • --time-style=STYL - Format czasu (full-iso, long-iso, iso, locale)

Filtrowanie

  • -B, --ignore-backups - Nie wyświetla plików kończących się na ~
  • -I, --ignore=WZORZEC - Nie wyświetla plików pasujących do wzorca
  • --hide=WZORZEC - Ukrywa pliki pasujące do wzorca
  • -X - Sortuj alfabetycznie według rozszerzenia

Informacje

  • --version - Wyświetla wersję programu
  • --help - Wyświetla pomoc

Przykłady użycia

mv - Przenoszenie i zmiana nazwy plików

Opis

Komenda mv (move) przenosi pliki i katalogi lub zmienia ich nazwy. W przeciwieństwie do cp, usuwa plik źródłowy po przeniesieniu.

Składnia

mv [OPCJE] ŹRÓDŁO CEL
mv [OPCJE] ŹRÓDŁO... KATALOG

Parametry i opcje

  • -i, --interactive - Pytaj przed nadpisaniem
  • -f, --force - Wymuś nadpisanie bez pytania
  • -n, --no-clobber - Nie nadpisuj istniejących plików
  • -v, --verbose - Pokazuj co jest przenoszone
  • -u, --update - Przenoś tylko nowsze pliki
  • -b - Twórz backup nadpisywanych plików
  • -t, --target-directory=DIR - Przenoś do katalogu DIR
  • --backup[=CONTROL] - Twórz backup (simple, numbered, existing)
  • -S, --suffix=SUFFIX - Nadpisz suffix backupu

Przykłady użycia

Podstawowe użycie

  1. Zmień nazwę pliku
mv stara-nazwa.txt nowa-nazwa.txt
  1. Przenieś plik do katalogu
mv plik.txt /home/user/Documents/
  1. Przenieś wiele plików
mv plik1.txt plik2.txt plik3.txt /backup/
  1. Przenieś z potwierdzeniem
mv -i plik.txt /backup/
  1. Przenieś z informacją
mv -v *.txt /backup/
  1. Zmień nazwę katalogu
mv stary-katalog nowy-katalog
  1. Przenieś katalog
mv katalog /nowa/lokalizacja/
  1. Nie nadpisuj
mv -n plik.txt /backup/
  1. Przenoś tylko nowsze
mv -u source/*.txt dest/
  1. Z backupem
mv -b plik.txt istniejacy.txt

Zmiana nazw

  1. Dodaj prefix
for f in *.txt; do mv "$f" "prefix-$f"; done
  1. Dodaj suffix
for f in *.jpg; do mv "$f" "${f%.jpg}-backup.jpg"; done
  1. Zmień rozszerzenie
for f in *.jpeg; do mv "$f" "${f%.jpeg}.jpg"; done
  1. Uppercase do lowercase
for f in *; do mv "$f" "$(echo $f | tr '[:upper:]' '[:lower:]')"; done
  1. Usuń spacje
for f in *\ *; do mv "$f" "${f// /_}"; done
  1. Numeruj pliki
i=1; for f in *.txt; do mv "$f" "file_$(printf %03d $i).txt"; ((i++)); done
  1. Dodaj datę
for f in *.log; do mv "$f" "${f%.log}-$(date +%Y%m%d).log"; done

Z wzorcami

  1. Przenieś wszystkie .txt
mv *.txt /backup/
  1. Przenieś różne rozszerzenia
mv *.{txt,pdf,doc} /backup/
  1. Przenieś z wykluczeniem
shopt -s extglob
mv !(*.tmp) /backup/

Kombinacje

  1. Przenieś z find
find . -name "*.log" -exec mv {} /logs/ \;
  1. Przenieś starsze niż 30 dni
find . -type f -mtime +30 -exec mv {} /archive/ \;
  1. Przenieś największe pliki
ls -S | head -10 | xargs -I {} mv {} /archive/
  1. Z rename (perl)
rename 's/\.jpeg$/.jpg/' *.jpeg
  1. Atomic rename
mv config.new config && systemctl restart service

Częste scenariusze

Organizacja plików

mv *.jpg ~/Pictures/
mv *.mp3 ~/Music/
mv *.pdf ~/Documents/

Rotacja logów

mv app.log app.log.old
touch app.log

Deployment

mv app-new app
mv config-new.yml config.yml

Wskazówki

  1. mv nie wymaga -r - Automatycznie przenosi katalogi
  2. Atomowa operacja - W tym samym systemie plików
  3. Zmiana nazwy - mv służy też do zmiany nazw
  4. Nie działa między systemami - Wtedy kopiuje i usuwa
  5. Szybkie - Zmienia tylko wpis w katalogu (ten sam filesystem)

pwd - Wyświetl bieżący katalog

Opis

Komenda pwd (print working directory) wyświetla pełną ścieżkę bezwzględną do bieżącego katalogu roboczego. Jest dostępna zarówno jako wbudowana komenda powłoki, jak i jako zewnętrzny program.

Składnia

pwd [OPCJE]

Parametry i opcje

Opcje

  • -L, --logical - Wyświetla logiczną ścieżkę (domyślne), podąża za linkami symbolicznymi
  • -P, --physical - Wyświetla fizyczną ścieżkę, unika linków symbolicznych
  • --help - Wyświetla pomoc
  • --version - Wyświetla wersję programu

Przykłady użycia

Podstawowe użycie

  1. Wyświetl bieżący katalog
pwd
# Output: /home/user/Documents
  1. Wyświetl fizyczną ścieżkę (bez linków)
pwd -P
  1. Wyświetl logiczną ścieżkę (z linkami)
pwd -L
  1. W skrypcie bash
#!/bin/bash
echo "Obecnie jesteś w: $(pwd)"
  1. Przypisz do zmiennej
CURRENT_DIR=$(pwd)
echo $CURRENT_DIR

Porównanie -L i -P

  1. Z linkiem symbolicznym (logical)
cd /tmp/link-to-home
pwd -L
# Output: /tmp/link-to-home
  1. Z linkiem symbolicznym (physical)
cd /tmp/link-to-home
pwd -P
# Output: /home/user (rzeczywista lokalizacja)
  1. Domyślne zachowanie
pwd
# Równoważne pwd -L

Użycie w skryptach

  1. Określ ścieżkę bazową skryptu
#!/bin/bash
SCRIPT_DIR=$(pwd)
echo "Skrypt uruchomiony z: $SCRIPT_DIR"
  1. Bezpieczna ścieżka do skryptu
#!/bin/bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
echo "Katalog skryptu: $SCRIPT_DIR"
  1. Zapisz i przywróć katalog
ORIGINAL_DIR=$(pwd)
cd /tmp
# wykonaj operacje
cd "$ORIGINAL_DIR"

Kombinacje z innymi komendami

  1. Utwórz plik w bieżącym katalogu z pełną ścieżką
touch "$(pwd)/nowy_plik.txt"
  1. Wyświetl pełną ścieżkę pliku
echo "$(pwd)/plik.txt"
  1. Kopiuj używając pełnej ścieżki
cp plik.txt $(pwd)/backup/
  1. Listuj z pełnymi ścieżkami
ls -d $(pwd)/*
  1. Znajdź pliki z pełną ścieżką
find $(pwd) -name "*.txt"
  1. W pętli for

    echo "$(pwd)/$file"
done
  1. Z xargs
ls | xargs -I {} echo "$(pwd)/{}"

Zaawansowane użycie

  1. Sprawdź czy jesteś w konkretnym katalogu

    echo "Jesteś w katalogu domowym"
fi
  1. Porównaj katalogi

Wskazówki i pułapki

  1. pwd vs $PWD - Zmienna $PWD jest szybsza niż wywołanie komendy pwd
  2. Linki symboliczne - Domyślnie pwd pokazuje logiczną ścieżkę (z linkami)

  4. Przestrzeń nazw - W kontenerach pwd może pokazywać inną ścieżkę niż w hoście

Historia

Komenda pwd jest częścią standardu POSIX i jest dostępna we wszystkich systemach uniksowych od wczesnych lat 70. XX wieku. Wbudowana wersja w powłokach (bash, ksh, zsh) jest zwykle szybsza niż zewnętrzny program /bin/pwd.

rm - Usuwanie plików i katalogów

Opis

Komenda rm (remove) usuwa pliki i katalogi z systemu plików. Jest to potężne i potencjalnie niebezpieczne narzędzie - usunięte pliki nie trafiają do kosza i zwykle nie mogą być odzyskane.

Składnia

rm [OPCJE] PLIK...

Parametry i opcje

Podstawowe opcje

  • -f, --force - Wymusza usunięcie, ignoruje nieistniejące pliki, nie pyta o potwierdzenie
  • -i - Pyta o potwierdzenie przed usunięciem każdego pliku
  • -I - Pyta raz przed usunięciem więcej niż 3 plików lub przy rekursywnym usuwaniu
  • -r, -R, --recursive - Usuwa katalogi i ich zawartość rekursywnie
  • -d, --dir - Usuwa puste katalogi
  • -v, --verbose - Wyświetla co jest usuwane
  • --preserve-root - Nie usuwa katalogu / (domyślne)
  • --no-preserve-root - Pozwala usunąć katalog /
  • --one-file-system - Nie przekracza granic systemów plików przy rekursji

Dodatkowe opcje

  • --interactive[=WHEN] - Pyta o potwierdzenie: never, once, always
  • -v, --verbose - Pokazuje co jest usuwane

Przykłady użycia

Podstawowe użycie

  1. Usuń pojedynczy plik
rm plik.txt
  1. Usuń wiele plików
rm plik1.txt plik2.txt plik3.txt
  1. Usuń wszystkie pliki .txt

  1. Usuń z potwierdzeniem

# Zapyta: usuń regularny plik 'plik.txt'? (y/n)
  1. Wymuś usunięcie (bez pytań)
rm -f plik.txt
  1. Usuń z informacją
rm -v plik.txt

Usuwanie katalogów

  1. Usuń pusty katalog

  1. Usuń katalog z zawartością
rm -r katalog
  1. Wymuś rekursywne usunięcie

# UWAGA: Bardzo niebezpieczne!
  1. Rekursywne z informacją

  1. Potwierdź przed dużym usunięciem

rmdir - Usuwanie pustych katalogów

Opis

Komenda rmdir (remove directory) usuwa puste katalogi. W przeciwieństwie do rm -r, rmdir działa tylko na pustych katalogach, co czyni ją bezpieczniejszym narzędziem.

Składnia

Parametry i opcje

Opcje

  • -p, --parents - Usuwa katalog wraz z katalogami nadrzędnymi, jeśli stają się puste
  • -v, --verbose - Wyświetla komunikat dla każdego usuniętego katalogu
  • --ignore-fail-on-non-empty - Ignoruje błędy wynikające z niepustych katalogów
  • --help - Wyświetla pomoc
  • --version - Wyświetla wersję programu

sed - Stream editor

Opis

sed (stream editor) to edytor strumieniowy do parsowania i transformacji tekstu. Czyta tekst linia po linii, wykonuje na nim operacje i wypisuje wynik.

Składnia

sed [OPCJE] 'KOMENDA' [PLIK...]
sed [OPCJE] -f SKRYPT [PLIK...]

Parametry

  • -n, --quiet, --silent - Ukrywa automatyczne drukowanie
  • -e SKRYPT - Dodaje skrypt do wykonania
  • -f PLIK - Dodaje komendy z pliku
  • -i[SUFFIX] - Edytuj plik w miejscu (opcjonalny backup)
  • -r, -E - Używa rozszerzonych wyrażeń regularnych
  • --follow-symlinks - Podąża za linkami symbolicznymi z -i

Komendy sed

Podstawienie: - s/wzorzec/zamiana/ - Podstawienie (pierwszy) - s/wzorzec/zamiana/g - Global (wszystkie) - s/wzorzec/zamiana/i - Case insensitive - s/wzorzec/zamiana/p - Drukuj jeśli zmiana - s/wzorzec/zamiana/2 - Tylko drugie wystąpienie

Usuwanie: - d - Usuń linie - /wzorzec/d - Usuń dopasowane linie - 5d - Usuń linię 5 - 1,5d - Usuń linie 1-5

Wypisywanie: - p - Drukuj linię - /wzorzec/p - Drukuj dopasowane - -n '5p' - Drukuj tylko linię 5

Zakresy: - 1,5 - Linie od 1 do 5 - /start/,/end/ - Od wzorca do wzorca - $ - Ostatnia linia

Przykłady użycia

Podstawienie

  1. Zamień pierwsze wystąpienie
sed 's/old/new/' file.txt
  1. Zamień wszystkie
sed 's/old/new/g' file.txt
  1. Case insensitive
sed 's/error/ERROR/i' log.txt
  1. Zamień w konkretnej linii
sed '5s/old/new/' file.txt
  1. Zamień w zakresie
sed '1,10s/old/new/g' file.txt
  1. Używając separatora innego niż /
sed 's|/old/path|/new/path|g' file.txt
  1. Zamień z backreference
sed 's/\([0-9]\{3\}\)/(\1)/' phone.txt

Usuwanie

  1. Usuń puste linie
sed '/^$/d' file.txt
  1. Usuń linie z komentarzami
sed '/^#/d' config.txt
  1. Usuń konkretną linię
sed '5d' file.txt
  1. Usuń zakres
sed '1,10d' file.txt
  1. Usuń od wzorca do końca
sed '/START/,$d' file.txt

Wypisywanie

  1. Drukuj tylko dopasowane
sed -n '/pattern/p' file.txt

umask - Maska uprawnień dla nowych plików

Opis

Komenda umask (user file-creation mode mask) ustawia domyślną maskę uprawnień dla nowo tworzonych plików i katalogów. Określa które uprawnienia NIE będą nadawane.

Składnia

umask [OPCJE] [MASKA]

Parametry i opcje

  • -S - Wyświetla maskę w notacji symbolicznej
  • -p - Wyświetla w formacie do ponownego użycia

Jak działa umask

Podstawowe zasady

  • Domyślne uprawnienia plików: 666 (rw-rw-rw-)
  • Domyślne uprawnienia katalogów: 777 (rwxrwxrwx)
  • umask odejmuje uprawnienia od domyślnych

Wzór

Końcowe uprawnienia = Domyślne - umask

Przykłady użycia

Podstawowe użycie

  1. Wyświetl obecną maskę
umask
# Output: 0022 (typowa wartość)
  1. Wyświetl w notacji symbolicznej
umask -S
# Output: u=rwx,g=rx,o=rx
  1. Ustaw umask 022
umask 022
# Pliki: 644 (rw-r--r--)
# Katalogi: 755 (rwxr-xr-x)
  1. Ustaw umask 002
umask 002
# Pliki: 664 (rw-rw-r--)
# Katalogi: 775 (rwxrwxr-x)
  1. Restrykcyjny umask 077
umask 077
# Pliki: 600 (rw-------)
# Katalogi: 700 (rwx------)

Obliczanie uprawnień

  1. umask 022 dla plików
666 (domyślne)
-022 (umask)
----
644 (rw-r--r--)
  1. umask 022 dla katalogów
777 (domyślne)
-022 (umask)
----
755 (rwxr-xr-x)
  1. umask 027
# Pliki: 640 (rw-r-----)
# Katalogi: 750 (rwxr-x---)

Różne wartości umask

  1. Bardzo restrykcyjny (077)
umask 077
touch test.txt
ls -l test.txt
# -rw------- (tylko właściciel)
  1. Wspólna grupa (002)
umask 002
mkdir projekt
ls -ld projekt
# drwxrwxr-x (grupa może pisać)
  1. Standardowy (022)
umask 022
touch dokument.txt
# -rw-r--r-- (grupa i inni tylko czytają)
  1. Bardzo otwarty (000) - NIEBEZPIECZNY
umask 000
touch public.txt
# -rw-rw-rw- (wszyscy mogą pisać)

W skryptach

  1. Tymczasowa zmiana umask
#!/bin/bash
OLD_UMASK=$(umask)
umask 077
# ... operacje na plikach ...
umask $OLD_UMASK
  1. Umask dla sekcji skryptu
(
    umask 027
    touch prywatny.txt
)
# umask powraca do poprzedniej wartości
  1. Warunkowy umask
if [ "$USER" = "admin" ]; then
    umask 022
else
    umask 077
fi

  1. Production environment

  1. Personal workspace

  • Katalogi: 775 (rwxrwxr-x)
  • Użycie: Współpraca w grupie

  • Pliki: 640 (rw-r—–)
  • Katalogi: 750 (rwxr-x—)
  • Użycie: Środowiska produkcyjne

umask 077 (Prywatny)

  • Pliki: 600 (rw——-)
  • Katalogi: 700 (rwx——)
  • Użycie: Dane osobiste, backupy
