Struktury kontrolne i sterujące w Bash - Kompletny przewodnik

Kompleksowy przewodnik po strukturach kontrolnych i sterujących w powłoce Bash (Bourne Again Shell). Dokument zawiera szczegółowe wyjaśnienia składni, zasad działania i praktycznych przykładów użycia.

Spis treści

  1. Instrukcja warunkowa if-then-else-fi
  2. Instrukcja warunkowa if-then-elif-else-fi
  3. Pętla for-in-do-done
  4. Pętla while-do-done
  5. Pętla until-do-done
  6. Instrukcja wyboru case-in-esac
  7. Instrukcje sterujące pętlami
  8. Operatory porównań

1. Instrukcja warunkowa if-then-else-fi

Składnia podstawowa

if warunek
then
    polecenia_gdy_prawda
else
    polecenia_gdy_fałsz
fi

Składnia w jednej linii

if warunek; then polecenia_gdy_prawda; else polecenia_gdy_fałsz; fi

Opis elementów

Element Opis
if Słowo kluczowe rozpoczynające instrukcję warunkową
warunek Wyrażenie zwracające kod wyjścia (0 = prawda, ≠0 = fałsz)
then Słowo kluczowe poprzedzające blok poleceń wykonywanych gdy warunek jest prawdziwy
else Słowo kluczowe poprzedzające blok poleceń wykonywanych gdy warunek jest fałszywy (opcjonalne)
fi Słowo kluczowe zamykające instrukcję (if pisane wspak)

Jak działa

  1. if - rozpoczyna blok warunkowy
  2. warunek - wyrażenie, które zwraca kod wyjścia (0 = prawda, ≠0 = fałsz)
  3. then - rozpoczyna blok poleceń wykonywanych gdy warunek jest prawdziwy
  4. else - (opcjonalne) rozpoczyna blok poleceń wykonywanych gdy warunek jest fałszywy
  5. fi - kończy blok warunkowy

Przykłady

Prosty if bez else

#!/bin/bash
liczba=10

if [ $liczba -gt 5 ]
then
    echo "Liczba jest większa od 5"
fi

if z else

#!/bin/bash
wiek=16

if [ $wiek -ge 18 ]
then
    echo "Jesteś pełnoletni"
else
    echo "Jesteś niepełnoletni"
fi

Sprawdzanie istnienia pliku

#!/bin/bash
plik="/etc/passwd"

if [ -f "$plik" ]
then
    echo "Plik $plik istnieje"
else
    echo "Plik $plik nie istnieje"
fi

Sprawdzanie czy katalog istnieje

#!/bin/bash
katalog="/home/user/dokumenty"

if [ -d "$katalog" ]; then
    echo "Katalog istnieje"
    ls -la "$katalog"
else
    echo "Katalog nie istnieje, tworzę..."
    mkdir -p "$katalog"
fi

Porównywanie ciągów znaków

#!/bin/bash
imie="Anna"

if [ "$imie" = "Anna" ]
then
    echo "Cześć Anna!"
else
    echo "Nie znam cię"
fi

Sprawdzanie argumentu

#!/bin/bash
if [ -z "$1" ]
then
    echo "Nie podano argumentu!"
    exit 1
fi
echo "Argument: $1"

Różnice między [ ], [[ ]] i (( ))

[ ] - test POSIX, zgodny ze wszystkimi powłokami

if [ "$a" = "$b" ]; then 
    echo "równe"
fi

[[ ]] - rozszerzona składnia Bash, więcej możliwości

# Pattern matching z wildcards
if [[ "$tekst" == *"wzorzec"* ]]
then
    echo "Tekst zawiera 'wzorzec'"
fi

# Wyrażenia regularne
if [[ "$tekst" =~ ^[0-9]+$ ]]; then 
    echo "to liczba"
fi

# Nie wymaga cytowania zmiennych (ale nadal zalecane)
if [[ $zmienna == wartość ]]
then
    echo "Równe"
fi

(( )) - operacje arytmetyczne

if (( liczba > 5 )); then 
    echo "większa od 5"
fi

if (( liczba % 2 == 0 )); then 
    echo "parzysta"
fi

2. Instrukcja warunkowa if-then-elif-else-fi

Składnia

if warunek1
then
    polecenia_gdy_warunek1_prawdziwy
elif warunek2
then
    polecenia_gdy_warunek2_prawdziwy
elif warunek3
then
    polecenia_gdy_warunek3_prawdziwy
else
    polecenia_gdy_wszystkie_warunki_fałszywe
fi

Opis elementów

Element Opis
elif Skrót od “else if” - sprawdza kolejny warunek jeśli poprzednie były fałszywe

Jak działa

  1. Sprawdzany jest warunek1
  2. Jeśli prawdziwy → wykonaj polecenia i zakończ blok
  3. Jeśli fałszywy → sprawdź warunek2
  4. Proces powtarza się dla kolejnych elif
  5. Jeśli żaden warunek nie jest prawdziwy → wykonaj blok else

Przykłady

Oceny szkolne

#!/bin/bash
punkty=75

if [ $punkty -ge 90 ]
then
    echo "Ocena: 5 (bardzo dobry)"
elif [ $punkty -ge 75 ]
then
    echo "Ocena: 4 (dobry)"
elif [ $punkty -ge 60 ]
then
    echo "Ocena: 3 (dostateczny)"
elif [ $punkty -ge 50 ]
then
    echo "Ocena: 2 (dopuszczający)"
else
    echo "Ocena: 1 (niedostateczny)"
fi

Oceny z argumentem

Sprawdzanie typu pliku

#!/bin/bash

if [ -z "$sciezka" ]

elif [ -c "$sciezka" ]

elif [ -e "$sciezka" ]
then
    echo "$sciezka istnieje, ale nie jest plikiem ani katalogiem"
else
    echo "$sciezka nie istnieje"
fi

Porównywanie liczb

3. Pętla for-in-do-done

Składnia podstawowa

for zmienna in lista_wartości
do
    polecenia
done

Składnia w jednej linii

for zmienna in lista; do polecenia; done

Opis elementów

Element Opis
for Słowo kluczowe rozpoczynające pętlę
zmienna Nazwa zmiennej przechowującej aktualną wartość z listy
in Słowo kluczowe oddzielające zmienną od listy
lista_wartości Elementy do iteracji (rozdzielone spacjami lub wygenerowane)
do Słowo kluczowe rozpoczynające blok poleceń
done Słowo kluczowe zamykające pętlę

Jak działa

  1. for - rozpoczyna pętlę
  2. zmienna - nazwa zmiennej, która przyjmuje kolejne wartości
  3. in - słowo kluczowe oddzielające zmienną od listy
  4. lista_wartości - elementy, po których iteruje pętla
  5. do - rozpoczyna blok poleceń
  6. done - kończy pętlę

Przykłady

Iteracja po liście słów

#!/bin/bash
for owoc in jabłko gruszka śliwka wiśnia
do
    echo "Owoc: $owoc"
done

Wynik:

Owoc: jabłko
Owoc: gruszka
Owoc: śliwka
Owoc: wiśnia

Iteracja po plikach

#!/bin/bash
for plik in *.txt
do
    echo "Przetwarzam: $plik"
    wc -l "$plik"
done

Iteracja po zakresie liczb

#!/bin/bash
# Użycie {start..koniec}
for i in {1..5}
do
    echo "Numer: $i"
done

# Z krokiem {start..koniec..krok}
for i in {0..10..2}
do
    echo "Parzysta: $i"
done

# Zakres dziesiątek
for i in {0..100..10}
do
    echo "Dziesiątki: $i"
done

Klasyczna pętla for (styl C)

#!/bin/bash
for ((i=0; i<5; i++))
do
    echo "Indeks: $i"
done

# Lub w jednej linii
for ((i=0; i<5; i++)); do echo "Indeks: $i"; done

# Pętla z warunkiem złożonym
for ((i=0; i<10; i++))
do
    echo "Indeks: $i"
done

Iteracja po argumentach skryptu

#!/bin/bash
echo "Przekazane argumenty:"
for arg in "$@"
do
    echo "- $arg"
done

# Wersja z licznikiem argumentów
echo "Przekazano $# argumentów:"
for arg in "$@"
do
    echo "  - $arg"
done

Iteracja po wynikach polecenia

#!/bin/bash
# Użycie podstawienia poleceń
for user in $(cat /etc/passwd | cut -d: -f1)
do
    echo "Użytkownik: $user"
done

# Bardziej efektywna wersja
for user in $(cut -d: -f1 /etc/passwd | head -5)
do
    echo "Użytkownik: $user"
done

Iteracja po tablicy

#!/bin/bash
kolory=("czerwony" "zielony" "niebieski" "żółty")

for kolor in "${kolory[@]}"
do
    echo "Kolor: $kolor"
done

for i in "${!kolory[@]}"
do
    echo "Indeks $i: ${kolory[$i]}"
done

Przetwarzanie plików w katalogu

#!/bin/bash
for plik in /var/log/*.log
do
    if [ -f "$plik" ]
    then
        rozmiar=$(du -h "$plik" | cut -f1)

Iteracja po liniach pliku (uwaga: dzieli po spacjach!)

#!/bin/bash
# UWAGA: for dzieli po spacjach, nie po liniach!
for linia in $(cat plik.txt)
do
    echo "Słowo: $linia"
done

# Dla prawdziwej iteracji po liniach użyj while

do
    echo "Linia: $linia"
done < plik.txt

Zagnieżdżone pętle

    do
        echo "i=$i, j=$j"

4. Pętla while-do-done

Składnia

while warunek
do
    polecenia
done

Opis elementów

Element Opis
while Słowo kluczowe rozpoczynające pętlę
warunek Wyrażenie sprawdzane przed każdą iteracją (pętla działa dopóki warunek jest prawdziwy)
do Słowo kluczowe rozpoczynające blok poleceń
done Słowo kluczowe zamykające pętlę

Jak działa

  1. while - rozpoczyna pętlę
  2. warunek - sprawdzany przed każdą iteracją
  3. Jeśli warunek jest prawdziwy (kod wyjścia 0) → wykonaj polecenia
  4. Po wykonaniu poleceń → wróć do sprawdzenia warunku
  5. Jeśli warunek jest fałszywy → zakończ pętlę
  6. done - oznacza koniec bloku poleceń pętli

Przykłady

Prosta pętla while

#!/bin/bash
licznik=1

while [ $licznik -le 5 ]
do
    echo "Iteracja: $licznik"
    ((licznik++))
done

Odczytywanie pliku linia po linii

#!/bin/bash
while IFS= read -r linia
do
    echo "Linia: $linia"
done < plik.txt

# Z /etc/hosts
while IFS= read -r linia
do
    echo "Przeczytano: $linia"
done < /etc/hosts

Wyjaśnienie: - IFS= - zapobiega usuwaniu białych znaków na początku/końcu - read -r - zapobiega interpretacji backslash jako znaku ucieczki - < plik.txt - przekierowanie pliku jako wejście

Nieskończona pętla

#!/bin/bash
while true
do
    echo "Naciśnij Ctrl+C aby zakończyć"
    sleep 1
done

# Alternatywna składnia z :
while :
do
    echo "Działam..."
    sleep 1
done

Pętla nieskończona z warunkiem wyjścia

#!/bin/bash
while true
do
    read -p "Podaj liczbę (0 aby zakończyć): " liczba
    if [ "$liczba" = "0" ]; then
        echo "Do widzenia!"

    fi
    echo "Podałeś: $liczba"
done

5. Pętla until-do-done

Składnia

until warunek
do
    polecenia
done

Opis elementów

Element Opis
until Słowo kluczowe rozpoczynające pętlę
warunek Wyrażenie sprawdzane przed każdą iteracją (pętla działa dopóki warunek jest fałszywy)
do Słowo kluczowe rozpoczynające blok poleceń
done Słowo kluczowe zamykające pętlę

Jak działa

Pętla until jest odwrotnością pętli while:

  1. until - rozpoczyna pętlę
  2. warunek - sprawdzany przed każdą iteracją
  3. Jeśli warunek jest fałszywy (kod wyjścia ≠0) → wykonaj polecenia
  4. Po wykonaniu poleceń → wróć do sprawdzenia warunku
  5. Jeśli warunek jest prawdziwy → zakończ pętlę
  6. done - kończy pętlę

Różnica między while a until

  • while - wykonuje się dopóki warunek jest prawdziwy
  • until - wykonuje się dopóki warunek jest fałszywy (czyli do momentu gdy stanie się prawdziwy)
# Te dwie pętle są równoważne:
while [ $x -lt 10 ]; do ...; done
until [ $x -ge 10 ]; do ...; done

Przykłady

Podstawowy przykład

#!/bin/bash
licznik=1

until [ $licznik -gt 5 ]
do
    echo "Licznik: $licznik"
    ((licznik++))
done

Odliczanie

#!/bin/bash
licznik=10

until [ $licznik -eq 0 ]
do
    echo "Pozostało: $licznik"
    ((licznik--))
    sleep 1
done

echo "START!"

Oczekiwanie na połączenie sieciowe

#!/bin/bash
until ping -c 1 google.com &>/dev/null
do
    echo "Brak połączenia, ponawianie za 5 sekund..."
    sleep 5
done
echo "Połączenie nawiązane!"

Oczekiwanie na plik

#!/bin/bash
plik="/tmp/sygnał.txt"

echo "Oczekuję na utworzenie pliku $plik..."

until [ -f "$plik" ]
do
    echo "Plik jeszcze nie istnieje..."
    sleep 2
done

echo "Plik został utworzony!"
cat "$plik"

Oczekiwanie na zakończenie procesu

#!/bin/bash
pid=$1

until ! ps -p $pid &>/dev/null
do
    echo "Proces $pid nadal działa..."
    sleep 2
done
echo "Proces $pid zakończył się"

Oczekiwanie na serwer

#!/bin/bash
host="localhost"
port=8080

until nc -z "$host" "$port" 2>/dev/null
do
    echo "Czekam na serwer $host:$port..."
    sleep 1
done

echo "Serwer jest dostępny!"

Pobieranie poprawnych danych od użytkownika

#!/bin/bash
liczba=""

until [[ "$liczba" =~ ^[0-9]+$ ]]
do
    read -p "Podaj liczbę całkowitą: " liczba
    if [[ ! "$liczba" =~ ^[0-9]+$ ]]
    then
        echo "To nie jest liczba!"
    fi
done

echo "Podałeś liczbę: $liczba"

Próby połączenia z limitem

#!/bin/bash
max_prob=5
proba=0

until ping -c 1 google.com &>/dev/null || [ $proba -ge $max_prob ]
do
    ((proba++))
    echo "Próba $proba z $max_prob - brak połączenia"
    sleep 2
done

if [ $proba -ge $max_prob ]; then
    echo "Nie udało się połączyć po $max_prob próbach"
    exit 1
else
    echo "Połączenie nawiązane!"
fi

Porównanie while i until

#!/bin/bash
# Te dwie pętle robią to samo:

# Wersja while
i=1
while [ $i -le 5 ]; do
    echo "while: $i"
    ((i++))
done

# Wersja until
i=1
until [ $i -gt 5 ]; do
    echo "until: $i"
    ((i++))
done

6. Instrukcja wyboru case-in-esac

Składnia

case wyrażenie in
    wzorzec1)
        polecenia1
        ;;
    wzorzec2)
        polecenia2
        ;;
    wzorzec3 | wzorzec4)
        polecenia3
        ;;
    *)
        polecenia_domyślne
        ;;
esac

Opis elementów

Element Opis
case Słowo kluczowe rozpoczynające instrukcję wyboru
wyrażenie Wartość do dopasowania ze wzorcami
in Słowo kluczowe poprzedzające listę wzorców
wzorzec) Wzorzec do dopasowania (może zawierać znaki globbing/wildcards)
;; Zakończenie bloku poleceń dla danego wzorca (break)
;& Przejście do następnego bloku bez sprawdzania (fall-through)
;;& Kontynuacja sprawdzania kolejnych wzorców
*) Wzorzec domyślny (pasuje do wszystkiego)
esac Słowo kluczowe zamykające instrukcję (case pisane wspak)

Jak działa

  1. case - rozpoczyna instrukcję wyboru
  2. wyrażenie - wartość do dopasowania
  3. in - słowo kluczowe
  4. wzorzec) - wzorzec do dopasowania (może zawierać znaki globbing)
  5. polecenia - wykonywane gdy wzorzec pasuje
  6. ;; - kończy blok danego wzorca (obowiązkowe!)
  7. *) - wzorzec domyślny (pasuje do wszystkiego)
  8. esac - kończy instrukcję case

Znaki specjalne we wzorcach

Znak Znaczenie
* Dowolny ciąg znaków (w tym pusty)
? Dokładnie jeden dowolny znak
[abc] Jeden znak z podanego zbioru
[a-z] Jeden znak z zakresu
[!abc] Jeden znak spoza zbioru
\| Alternatywa (lub)

Wzorce w case - szczegóły

case $zmienna in
    tekst)          # Dokładne dopasowanie
    [Tt]ekst)       # Dopasowanie z alternatywą znaków
    tekst*)         # Zaczyna się od "tekst"
    *tekst)         # Kończy się na "tekst"
    *tekst*)        # Zawiera "tekst"
    [0-9])          # Pojedyncza cyfra
    [a-zA-Z])       # Pojedyncza litera
    ???)            # Dokładnie 3 znaki
    a|b|c)          # "a" LUB "b" LUB "c"
    *)              # Wszystko inne (domyślny)
esac

8. Operatory porównań

Operatory dla liczb (w [ ] i [[ ]])

Operator Znaczenie Przykład
-eq równe (equal) [ $a -eq $b ]
-ne różne (not equal) [ $a -ne $b ]
-lt mniejsze (less than) [ $a -lt $b ]
-le mniejsze lub równe (less or equal) [ $a -le $b ]
-gt większe (greater than) [ $a -gt $b ]
-ge większe lub równe (greater or equal) [ $a -ge $b ] 
if [ $a -eq $b ]; then echo "równe"; fi
if [ $a -lt $b ]; then echo "a < b"; fi
if [ $a -ne $b ]; then echo "różne"; fi

Operatory dla liczb (w (( )))

W podwójnych nawiasach okrągłych możemy używać operatorów matematycznych w stylu C:

if (( a == b )); then echo "równe"; fi
if (( a != b )); then echo "różne"; fi
if (( a < b ));  then echo "a < b"; fi
if (( a <= b )); then echo "a <= b"; fi
if (( a > b ));  then echo "a > b"; fi
if (( a >= b )); then echo "a >= b"; fi

# Można też używać operacji arytmetycznych
if (( a + b > 10 )); then echo "suma > 10"; fi
if (( a % 2 == 0 )); then echo "parzysta"; fi

Operatory dla ciągów znaków

Operator Znaczenie Przykład
= równe [ "$a" = "$b" ]
== równe (w [[ ]]) [[ "$a" == "$b" ]]
!= różne [ "$a" != "$b" ]
< mniejsze leksykograficznie [[ "$a" < "$b" ]]
> większe leksykograficznie [[ "$a" > "$b" ]]
-z pusty ciąg (zero length) [ -z "$a" ]
-n niepusty ciąg (non-zero) [ -n "$a" ] 
if [ "$a" = "$b" ]; then echo "równe"; fi
if [ -z "$a" ]; then echo "pusty"; fi
if [ -n "$a" ]; then echo "niepusty"; fi

# W [[ ]] można używać pattern matching
if [[ "$tekst" == *"wzorzec"* ]]; then echo "zawiera wzorzec"; fi
if [[ "$email" =~ ^[a-zA-Z0-9]+@[a-zA-Z0-9]+\.[a-z]+$ ]]; then echo "poprawny email"; fi

Testy plikowe

Operator Znaczenie
-e plik istnieje (exists)
-f to zwykły plik (file)
-d to katalog (directory)
-L to link symboliczny
-r można czytać (readable)
-w można pisać (writable)
-x można wykonać (executable)
-s plik nie jest pusty (size > 0)
-O użytkownik jest właścicielem
-G grupa jest właścicielem
-h to link symboliczny (jak -L)
-b urządzenie blokowe
-c urządzenie znakowe
-p nazwany potok (pipe)
-S gniazdo (socket) 
if [ -f "$plik" ]; then echo "to plik"; fi
if [ -d "$katalog" ]; then echo "to katalog"; fi
if [ -r "$plik" ] && [ -w "$plik" ]; then echo "r/w"; fi
if [ -x "$skrypt" ]; then echo "wykonywalny"; fi
if [ -s "$plik" ]; then echo "niepusty"; fi

Porównania plikowe

Operator Znaczenie
plik1 -nt plik2 plik1 nowszy niż plik2 (newer than)
plik1 -ot plik2 plik1 starszy niż plik2 (older than)
plik1 -ef plik2 pliki mają ten sam inode (equal file)

Operatory logiczne

W pojedynczych nawiasach [ ]

# AND
if [ warunek1 ] && [ warunek2 ]; then
    echo "oba prawdziwe"
fi

# LUB
if [ warunek1 ] || [ warunek2 ]; then
    echo "przynajmniej jeden prawdziwy"
fi

# Negacja
if [ ! warunek ]; then
    echo "warunek fałszywy"
fi

# Alternatywna składnia AND/OR (starsza, mniej zalecana)
if [ warunek1 -a warunek2 ]; then
    echo "oba prawdziwe"
fi

if [ warunek1 -o warunek2 ]; then
    echo "przynajmniej jeden prawdziwy"
fi

W podwójnych nawiasach [[ ]]

# AND
if [[ warunek1 && warunek2 ]]; then
    echo "oba prawdziwe"
fi

# OR
if [[ warunek1 || warunek2 ]]; then
    echo "przynajmniej jeden prawdziwy"
fi

# Negacja
if [[ ! warunek ]]; then
    echo "warunek fałszywy"
fi

9. Najczęstsze błędy

1. Brak spacji w nawiasach

# ŹLE - brak spacji wewnątrz [ ]
if [$a -eq $b]; then
    echo "równe"
fi

# DOBRZE - spacje wymagane
if [ $a -eq $b ]; then
    echo "równe"
fi

Wyjaśnienie: [ to właściwie polecenie (alias dla test), więc musi być oddzielone spacjami.

2. Brak cudzysłowów przy zmiennych

# ŹLE - błąd gdy zmienna jest pusta lub zawiera spacje
if [ $zmienna = "tekst" ]; then
    echo "OK"
fi

# DOBRZE - zawsze cytuj zmienne
if [ "$zmienna" = "tekst" ]; then
    echo "OK"
fi

Dlaczego: Gdy $zmienna jest pusta, pierwszy przykład staje się: [ = "tekst" ] co jest błędem składni.

3. Używanie = zamiast -eq dla liczb

# ŹLE - porównanie tekstowe, nie numeryczne
if [ 10 = 9 ]; then  
    echo "równe"
fi

# To zadziała, ale porównuje stringi:
if [ "10" = "10" ]; then echo "równe"; fi  # OK, bo "10" == "10"
if [ "10" = "010" ]; then echo "równe"; fi  # Nie wykona, bo "10" != "010"

# DOBRZE - porównanie numeryczne
if [ 10 -eq 10 ]; then
    echo "równe"
fi
if [ 10 -eq 010 ]; then  # OK, oba reprezentują 10
    echo "równe"
fi

4. Brak then lub do

# ŹLE - brak then
if [ -f plik ]
    echo "istnieje"
fi

# DOBRZE - z then
if [ -f plik ]; then
    echo "istnieje"
fi

# ŹLE - brak do
for i in {1..5}
    echo $i
done

# DOBRZE - z do
for i in {1..5}; do
    echo $i
done

5. Brak fi, done lub esac

# ŹLE - brak zamknięcia
if [ -f plik ]; then
    echo "OK"
# <- brak fi

# DOBRZE - zawsze zamykaj struktury
if [ -f plik ]; then
    echo "OK"
fi

# ŹLE - brak done
for i in {1..5}; do
    echo $i
# <- brak done

# DOBRZE
for i in {1..5}; do
    echo $i
done

6. Używanie == w [ ] (nieprzenośne)

# Może nie działać wszędzie (nie jest POSIX)
if [ "$a" == "$b" ]; then
    echo "równe"
fi

# Przenośne (POSIX)
if [ "$a" = "$b" ]; then
    echo "równe"
fi

# Lub użyj [[ ]] w Bash (zalecane)
if [[ "$a" == "$b" ]]; then
    echo "równe"
fi

7. Nieprawidłowe cytowanie w for

# ŹLE - dzieli ścieżki ze spacjami
for plik in $(ls /ścieżka/ze spacjami/)
do
    echo "$plik"
done

# DOBRZE - użyj globbing zamiast ls
for plik in /ścieżka/ze\ spacjami/*
do
    echo "$plik"
done

# Lub iteruj po liniach (jeśli naprawdę musisz używać ls)
ls /ścieżka/ | while IFS= read -r plik
do
    echo "$plik"
done

# ŹLE - brak ;;

# ŹLE - licznik nie został zainicjalizowany
while [ $licznik -lt 10 ]

Podsumowanie

Struktura Zastosowanie Składnia skrócona
if-then-else-fi Proste rozgałęzienie warunkowe if [ w ]; then p1; else p2; fi
if-elif-else-fi Wielokrotne warunki if [ w1 ]; then p1; elif [ w2 ]; then p2; fi
for-in-do-done Iteracja po liście wartości for i in lista; do p; done
while-do-done Pętla dopóki warunek prawdziwy while [ w ]; do p; done
until-do-done Pętla dopóki warunek fałszywy until [ w ]; do p; done
case-in-esac Wybór na podstawie wzorców case $v in w1) p1;; *) p2;; esac

Kluczowe zasady

  1. Zawsze cytuj zmienne w warunkach: "$zmienna" zamiast $zmienna
  2. Używaj [[ ]] zamiast [ ] gdy to możliwe (więcej funkcji, mniej błędów)
  3. Pamiętaj o ;; na końcu każdego bloku w case
  4. Unikaj nieskończonych pętli - zawsze upewnij się, że warunek może się zmienić
  5. Testuj skrypty z różnymi danymi wejściowymi
  6. Inicjalizuj zmienne przed użyciem
  7. Używaj -eq, -lt itp. dla liczb, = dla stringów
  8. Pamiętaj o spacjach wewnątrz [ ]
  9. Zamykaj struktury - każde if wymaga fi, każdy for/while wymaga done, każdy case wymaga esac
  10. Używaj break i continue do kontroli przepływu w pętlach

Szybki podgląd składni

# if
if [ warunek ]; then polecenia; else inne; fi

# if-elif
if [ w1 ]; then p1; elif [ w2 ]; then p2; else p3; fi

# for
for i in lista; do polecenia; done
for ((i=0; i<10; i++)); do polecenia; done

# while
while [ warunek ]; do polecenia; done

# until
until [ warunek ]; do polecenia; done

# case
case $zmienna in
    wzorzec1) polecenia;;
    wzorzec2) polecenia;;
    *) domyślne;;
esac

Wybór odpowiedniej struktury

  • if - gdy masz 1-2 warunki do sprawdzenia
  • if-elif - gdy masz wiele warunków do sprawdzenia sekwencyjnie
  • case - gdy sprawdzasz wartość zmiennej względem wielu wzorców (czytelniejsze niż długie if-elif)
  • for - gdy znasz z góry elementy do przetworzenia
  • while - gdy nie wiesz ile iteracji będzie potrzebnych, ale znasz warunek zakończenia
  • until - jak while, ale logika odwrócona (rzadziej używane)
  • break - natychmiastowe wyjście z pętli
  • continue - pominięcie reszty iteracji i przejście do następnej

Zalecane praktyki

  1. Komentuj kod - wyjaśnij co robi każda ważna sekcja
  2. Używaj mówiących nazw zmiennych - liczba_prob zamiast n
  3. Sprawdzaj błędy - testuj istnienie plików przed użyciem
  4. Używaj funkcji - dziel długie skrypty na funkcje
  5. Loguj działania - szczególnie w skryptach produkcyjnych
  6. Obsługuj sygnały - trap do przechwytywania Ctrl+C itp.
  7. Waliduj dane wejściowe - sprawdzaj argumenty skryptu
  8. Używaj set -e - przerwij skrypt przy błędzie
  9. Używaj set -u - traktuj niezdefiniowane zmienne jako błąd
  10. Używaj ShellCheck - linter do sprawdzania skryptów bash

Dodatkowe zasoby

Przydatne polecenia do debugowania

# Włącz tryb debugowania
set -x  # Wyświetla każde wykonywane polecenie
set -v  # Wyświetla każdą linię przed wykonaniem

# Wyłącz tryb debugowania
set +x
set +v

# Debuguj konkretną sekcję
set -x
# ... kod do debugowania ...
set +x

# Tryb strict
set -euo pipefail
# -e: przerwij przy błędzie
# -u: traktuj niezdefiniowane zmienne jako błąd
# -o pipefail: zwróć kod błędu z pipeline jeśli jakiekolwiek polecenie się nie powiodło

Testowanie warunków w terminalu

# Szybkie testowanie warunków
test -f /etc/passwd && echo "plik istnieje"
[ -d /home ] && echo "katalog istnieje"
[[ "abc" == "abc" ]] && echo "równe"

# Sprawdzanie kodu wyjścia
echo $?  # Wyświetla kod wyjścia ostatniego polecenia (0 = sukces)
